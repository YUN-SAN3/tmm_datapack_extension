### 1. spawn_pos (玩家出生位置)

- **作用**: 回合结束后所有玩家在大厅的出生位置和朝向
- **pos**: [x, y, z] 坐标
- **yaw**: 水平朝向（角度，0° 为南，90° 为西）
- **pitch**: 垂直朝向（角度，0° 为水平，-90° 为向上看，90° 为向下看）

### 2. spectator_spawn_pos (观众出生位置)

- **作用**: 回合开始时观众的出生位置和朝向
- **pos**: [x, y, z] 坐标
- **yaw**: 水平朝向（角度）
- **pitch**: 垂直朝向（角度）

### 3. ready_area (准备区域)

- **作用**: 玩家必须在此区域内才能算作准备好开始游戏
- **minX/Y/Z**: 区域的最小坐标
- **maxX/Y/Z**: 区域的最大坐标
- **说明**: 这是一个立方体区域，玩家需要站在这个区域内点击准备按钮才算准备好

### 4. play_area_offset (游戏区域偏移)

- **作用**: 玩家从准备区传送到游戏区时的坐标偏移量
- **x/y/z**: 在对应轴上的偏移量
- **说明**: 这个偏移量会应用于所有玩家的传送坐标

### 5. play_area (游戏区域)

- **作用**: 玩家被绑定的游戏区域
- **minX/Y/Z**: 区域的最小坐标
- **maxX/Y/Z**: 区域的最大坐标
- **说明**: 玩家不能离开这个区域，如果跌落到 minY 值以下会被自动杀死

### 6. reset_template_area (重置模板区域)

- **作用**: 地图重置基础模板所在的区域
- **minX/Y/Z**: 区域的最小坐标
- **maxX/Y/Z**: 区域的最大坐标
- **说明**: 这个区域包含地图的原始状态，用于回合重置

### 7. reset_paste_area (重置粘贴区域)

- **作用**: 地图重置时模板应该粘贴到的区域
- **minX/Y/Z**: 区域的最小坐标
- **maxX/Y/Z**: 区域的最大坐标
- **说明**: 这是地图重置时，模板内容会被粘贴到的目标区域

## 使用方法

1. 将此数据包放置在 Minecraft 世界的 datapacks 文件夹中
2. 使用 `/reload` 命令或重启游戏以应用更改
3. 配置将在下次游戏开始时生效

## 注意事项

1. 坐标值需要根据你的实际地图设计进行调整
2. 确保游戏区域足够大，能够容纳所有玩家活动
3. 重置区域的模板区域和粘贴区域通常大小相同，但位置可以不同
4. 角度值使用度作为单位，而非弧度
